**Дата:** 2026-07-04

## 1. Контекст и цель

Сергей Дмитриев — DevOps-инженер с опытом работы в бигтехе (Wildberries, масштаб и трафик уровня миллионов пользователей). Переходит из найма в премиум-консалтинг по инфраструктуре.

Нужен брендбук, который:
- **Для живых дизайнеров** — читаемый веб-сайт с принципами, примерами, do/don't галереями.
- **Для AI-агентов** — машиночитаемые токены (W3C JSON) и Markdown-описания голоса/тона, чтобы любой LLM мог генерировать контент в стиле Сергея.

Позиционирование: **«Инженер масштаба» по сути + премиум-подача стратегического консультанта.** Авторитет через реальный опыт масштабов, прямой тон без хайпа, дорогой визуал.

| Токен | HEX | Роль |
|---|---|---|
| `color.canvas.dark` | `#0A0A0B` | Фон страницы |
| `color.surface.dark` | `#101012` | Карточки, панели |
| `color.elevated.dark` | `#141416` | Приподнятые элементы |
| `color.border.dark` | `#1F1F23` | Тонкие границы |

| `color.text-mute.dark` | `#71717A` | Метки, подписи |

**Светлая тема (документы/PDF):**

| Токен | HEX | Роль |
|---|---|---|
| `color.canvas.light` | `#FAFAF9` | Фон (тёплый оффвайт) |
| `color.surface.light` | `#FFFFFF` | Карточки |
| `color.border.light` | `#E7E5E4` | Тонкие границы |
| `color.text.light` | `#1C1917` | Основной текст |
| `color.text-dim.light` | `#57534E` | Вторичный текст |
| `color.text-mute.light` | `#78716C` | Метки, подписи |

**Янтарная шкала (акцент, выровнена по Tailwind для узнаваемости):**

| Токен | HEX | Использование |
|---|---|---|
| `color.amber.900` | `#451A03` | Тёмные фоны-акценты |
| `color.amber.700` | `#92400E` | Глубокий янтарь, редко |
| `color.amber.600` | `#D97706` | Текстовый акцент на светлом фоне |
| `color.amber.500` | `#F59E0B` | **Базовый акцент**, UI, CTA |
| `color.amber.400` | `#FBBF24` | Bright, hover-состояния |

**Статусные цвета** (только для семантических состояний систем/процессов):

| Токен | HEX | Семантика |
|---|---|---|
| `color.status.success` | `#10B981` | Успех, healthy, running |
| `color.status.warning` | `#F97316` | Деградация, degraded |
| `color.status.error` | `#EF4444` | Ошибка, crash, down |
| `color.status.info` | `#3B82F6` | Информация, deploying |

**Правила цвета:**
- Янтарь — единственный UI-акцент. CTA, активные состояния, ссылки, ключевые числа.
- Янтарь никогда не используется как фон крупного блока.
- Янтарь `#F59E0B` не на белом фоне (низкий контраст). На светлом — `#D97706`.
- Статусные цвета — только для состояний систем, не для UI-акцентов.
- Все текстовые комбинации проходят WCAG AA (4.5:1), основные — AAA (7:1).

### 3.3 Логотип

**Состав:**
- **Монограмма** — квадрат со скруглением 11px (для размера 52px), янтарный градиент (135deg, `#FBBF24` → `#D97706`), внутри инициалы «СД» шрифтом Golos Text 800, цвет текста — Canvas темы.
- **Словесный знак** — «Дмитриев», Golos Text 700, letter-spacing 0.02em.

**Применения:**
- Полный: монограмма + словесный знак (по умолчанию).
- Только монограмма: аватары, favicon, мелкие места.
- Только словесный знак: узкие места, подписи.

**Правила:**
- Минимальный размер монограммы: 24px (цифровой), 12mm (печать).
- Clear space: равен высоте монограммы со всех сторон.
- Не拉伸ать, не менять цвета, не поворачивать, не добавлять тени.
- На тёмном фоне — градиент как есть. На светлом — градиент сохраняется, текст внутри тёмный.

### 4.2 Single source of truth: токены

`tokens/tokens.source.json` — единственный источник. Формат — [W3C Design Tokens](https://design-tokens.org/community-group/) (будущий стандарт, понимается AI-инструментами).

Скрипт `tokens/sync.js` читает JSON и генерирует `src/styles/tokens.css` с CSS custom properties. Запускается в `predev`/`prebuild` хуках npm, чтобы CSS всегда соответствовал JSON. Также проверяет, что все токены есть в обеих темах.

**Дизайнеры** видят красивый сайт, который читает CSS-переменные. **AI-агенты** читают JSON напрямую. **Один источник — два выхода.**

Пример структуры токена:
```json
{
  "color": {
    "amber": {
      "500": {
        "$value": "#F59E0B",
        "$type": "color",
        "$description": "Базовый акцент. CTA, активные состояния, ссылки."
      }
    }
  }
}
```

### 4.3 AGENTS.md

Короткий файл (≈1 страница) в корне репозитория — «system prompt» для любого AI-агента, работающего с брендом Сергея. Содержит:
- Одно предложение сути бренда.
- 5 ключевых принципов голоса и тона (без хайпа, по делу, метрики, etc.).
- Ссылку на `tokens/tokens.source.json` для точных значений.
- Ссылку на `src/content/01-foundation.md` для полного voice & tone.

Не дублирует токены — ссылается на JSON.

## 5. Структура контента по разделам

### 01 — Foundation
- Манифест «Инженер масштаба»: один абзац сути.
- Позиционирование: для кого, какую проблему решает, чем отличается.
- Принципы работы (5–7 пунктов): прагматизм над хайпом, метрики над словами, overengineering — зло, и т.д.
- Voice & tone: как звучит Сергей в текстах. Примеры «правильно/неправильно».
- Словарь: слова, которые использует и которых избегает.

### 02 — Logo
- Монограмма, словесный знак, полный вариант.
- Конструкция: пропорции, скругление, градиент.
- Clear space, минимальный размер.
- На тёмном / светлом фоне.
- Галерея don'ts (6–8 примеров: растянутый, с тенью, другого цвета, повёрнутый, с эффектами).

### 03 — Color
- Обе темы с swatch-картами.
- Янтарная шкала.
- Статусные цвета с DevOps-контекстом (`kubectl get pods` демо).
- Правила контраста, do/don't.

### 04 — Typography
- Golos Text и JetBrains Mono: образцы, начертания.
- Шкала размеров с живыми примерами.
- Пары заголовок/текст, иерархия.
- Правила: переносы, длина строки, моноширинный акцент.

### 05 — Tokens
- Документация формата W3C.
- Как читать JSON.
- Как подключить CSS-переменные.
- Примеры использования для дизайнеров и для AI-промптов.

### 06 — Application
- **Баннер для сайта/блога** — обложка статьи, hero-секция.
- **Карточка кейса** — структура: проблема → решение → метрики.
- **Слайд презентации** — заголовок + ключевой тезис + янтарный акцент.
- **Пост в соцсети** — Telegram-пост и LinkedIn-пост в голосе Сергея.
- **Шаблон PDF-кейса для клиентов** — структура документа на нескольких страницах.
- Каждый шаблон показан в обеих темах с пояснениями.

## 6. Технологический стек

- **Astro** — статический сайт, Markdown из коробки, компоненты для галерей, быстрая сборка.
- **Golos Text + JetBrains Mono** — Google Fonts (открытые лицензии, полная кириллица).
- **CSS custom properties** — для веб-страницы.
- **W3C Design Tokens JSON** — для AI-агентов и будущих дизайн-инструментов.
- **npm-скрипты** — `tokens/sync.js` для генерации CSS из JSON, без внешних зависимостей.
