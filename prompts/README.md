# Промпты · AI-готовность брендбука

Готовые промпт-шаблоны для генерации контента в стиле Сергея Дмитриева. Каждый промпт — это Markdown-файл с YAML frontmatter (метаданные + переменные) и телом промпта с встроенным бренд-контекстом.

## Структура

```
prompts/
├── _shared/                    # Общий бренд-контекст (не запускать отдельно)
│   ├── brand-context-ru.md     # Вставлен во все RU-промпты
│   └── brand-context-en.md     # Вставлен во все EN-промпты
├── text/                       # 6 промптов × 2 языка = 12 файлов
│   ├── telegram-post.{ru,en}.md
│   ├── linkedin-post.{ru,en}.md
│   ├── case-study.{ru,en}.md
│   ├── slide-thesis.{ru,en}.md
│   ├── email-newsletter.{ru,en}.md
│   └── blog-article.{ru,en}.md
├── visual/                     # 3 промпта × 2 языка = 6 файлов
│   ├── social-cover.{ru,en}.md
│   ├── og-image.{ru,en}.md
│   └── architecture-diagram.{ru,en}.md
└── ui/                         # 3 промпта × 2 языка = 6 файлов
    ├── ui-component.{ru,en}.md
    ├── landing-section.{ru,en}.md
    └── readme-docs.{ru,en}.md
```

**Всего: 24 промпта** (12 RU + 12 EN).

## Как пользоваться

### 1. Выбери промпт

По категории и языку. Например, для поста в Telegram на русском — `text/telegram-post.ru.md`.

### 2. Открой файл и заполни переменные

В YAML frontmatter указаны переменные (`variables:`), которые нужно подставить в теле промпта перед запуском. Например:

```yaml
variables:
  - name: topic
    description: Тема (кейс, тезис, новость)
  - name: metrics
    description: Конкретные числа (опционально)
```

В теле промпта найди `{{topic}}` и `{{metrics}}`, замени на свои значения.

### 3. Вставь промпт целиком в LLM

Скопируй всё содержимое файла (включая бренд-контекст вверху) в Claude / ChatGPT / Gemini / Cursor и т. д. Контекст уже встроен — больше ничего добавлять не нужно.

### 4. Проверь вывод по анти-паттерну

В каждом файле есть раздел «Анти-паттерн» — того, чего быть **не должно** (buzzwords, отсутствие метрик, оффбренд-стиль). Если вывод похож на анти-паттерн — попроси LLM переписать.

## Категории

### 📝 text/ — Текстовые LLM

Целевые инструменты: Claude, ChatGPT, Gemini.

| Промпт | Длина вывода | Назначение |
|---|---|---|
| `telegram-post` | 3–5 предложений | Пост о кейсе/тезисе |
| `linkedin-post` | 1–2 абзаца | Профессиональный пост (EN-фокус) |
| `case-study` | 500–800 слов | Полный кейс: проблема → решение → метрики |
| `slide-thesis` | 1–2 предложения | Тезис для слайда презентации |
| `email-newsletter` | 200–400 слов | Рассылка подписчикам/лидам |
| `blog-article` | 1000–1500 слов | Long-form статья для блога |

### 🎨 visual/ — Image gen и SVG

| Промпт | Целевые | Что генерирует |
|---|---|---|
| `social-cover` | Midjourney, DALL-E 3, SD XL | Обложка поста/статьи 1200×630 |
| `og-image` | Midjourney, DALL-E 3, Figma | Open Graph картинка для ссылок 1200×630 |
| `architecture-diagram` | Claude, ChatGPT, GPT-4 | SVG-схема архитектуры к кейсу |

**Важно:** AI image tools ненадёжно рендерят кириллицу. Для `social-cover` и `og-image` с русским текстом генерируй фон/мотив через AI, а текст накладывай в Figma. В промптах это отмечено.

### 💻 ui/ — Code/UI gen

Целевые инструменты: Cursor, ZCode, Claude Code, v0, Bolt.

| Промпт | Что генерирует |
|---|---|
| `ui-component` | Компонент (Astro/React/Vue) с готовыми токенами |
| `landing-section` | Секция лендинга: услуги, контакты, кейсы |
| `readme-docs` | README / docstring / документация в голосе Сергея |

UI-промпты включают полные таблицы CSS-токенов (`--amber-500`, `--canvas`, `--font-sans` и т. д.) — сгенерированный код сразу совместим с брендбуком.

## Бренд-голос (что отличает «наше» от «не нашего»)

| ✅ Do | ❌ Don't |
|---|---|
| Метрики: «p99 800→120 мс» | Buzzwords: «cutting-edge», «synergy» |
| Прагматизм: «Kubernetes ради Kubernetes — это долг» | Хайп: «революционное решение» |
| Прямой тон, без воды | Вода: «переосмыслили архитектуру» |
| Технические термины в English (RPS, MTTR, latency) | Калёные кальки: «переосмысление», «синергия» |

Полный бренд-контекст — в `_shared/brand-context-{ru,en}.md`.

## Когда обновлять промпты

- **Бренд-эволюция:** изменился голос/позиционирование → правь `_shared/brand-context-{ru,en}.md` (один раз), обновления подхватятся при следующем копировании в промпты.
- **Новый артефакт:** нужен промпт для нового типа контента (например, tweet-thread) — создай файл по тому же формату в нужной категории.
- **Токены изменились:** обновились цвета/шрифты → правь таблицу токенов в `ui/` промптах (только UI-промпты зависят от конкретных значений).

## Связанные файлы брендбука

- `AGENTS.md` (корень) — компактный AI-бриф.
- `tokens/tokens.source.json` — машиночитаемые значения (для UI-промптов).
- `src/pages/01-foundation.astro` — полный голос и тон.
- `src/pages/06-application.astro` — примеры «правильного» контента в каждом формате.
